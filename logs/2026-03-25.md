# Sessie-logboek — 2026-03-25

## Samenvatting
MacsyZones (open-source macOS window manager) is geforkt van `rohanrhu/MacsyZones` naar een eigen versie "MacsyZones Dev" met eigen bundle ID (`dev.ezwep.MacsyZonesDev`). De Pro-licentie, donatie-reminders en de originele update-koppeling zijn verwijderd. De auto-updater is gekoppeld aan de eigen repo `ezwep/MacsyZones`. Accessibility permissions worden automatisch gereset bij elke build.

## Doel
Een eigen ontwikkelversie van MacsyZones opzetten op basis van de originele open-source app, zodat de gebruiker verbeteringen kan gaan maken. De bestaande configuratie (layouts, instellingen) moest behouden blijven.

## Problemen & Oplossingen

| Probleem | Oplossing |
|----------|-----------|
| Accessibility permissions vervallen na rebuild (nieuw code signing identity) | Pre-build Run Script phase met `tccutil reset` die bij elke build de TCC entry reset |
| Xcode warning over Run Script phase zonder outputs | `alwaysOutOfDate = 1` flag toegevoegd — dit is gewenst gedrag |
| Runtime TCC reset vereist admin privileges (password prompt) | Verplaatst naar build-time Xcode script phase in plaats van runtime AppleScript |

## Beslissingen
- **Eigen bundle ID** (`dev.ezwep.MacsyZonesDev`) — zodat de Dev-versie naast de originele kan bestaan zonder conflicten
- **ProLock niet verwijderd maar geneutraliseerd** — `isPro = true` hardcoded, voorkomt compile errors door referenties in andere bestanden
- **DonationReminder.count() leeggemaakt** i.p.v. heel het bestand verwijderen — minimale wijzigingen, `States.swift` roept nog steeds `donationReminder.count()` aan
- **Updater behouden en omgeleid** — in plaats van verwijderen, gekoppeld aan eigen repo zodat toekomstige releases automatisch gedetecteerd worden
